Turvallisuus

  • Dokumenttitason pääsynhallinta: käyttäjä näkee vain dokumentit, joihin hänellä on oikeus
  • PII-poisto: automatisoi henkilötietojen poisto ennen indeksointia
  • Audit-loki: kaikki kyselyt ja vastaukset lokitetaan
ℹ️

Kriittinen oivallus: Yritys-RAGissa suurin haaste ei ole tekniikka vaan pääsynhallinta. Jos et ole tarkkana, käyttäjä voi kysellä dokumenteista, joihin hänellä ei ole pääsyä, RAG:n kautta.

Ihmiskontrolli ja eskalaatio

// Eskalaatio ihmiselle
async function handleConversation(input: string, context: Context) {
  const intent = await classifyIntent(input);

  if (intent.confidence < 0.7) {
    // En ole varma → kysy tarkennusta
    return askClarification();
  }

  if (intent.type === "complaint" || intent.sentiment === "angry") {
    // Valitus tai vihainen asiakas → heti ihmiselle
    return escalateToHuman(intent);
  }

  const response = await getAIResponse(input, context);
  return response;
}

5.3 Koodiavustaja (Copilot-tyyppinen)

Kontekstin keräys repo-tasolla

interface RepoContext {
  openFiles: string[];           // Tällä hetkellä avoimet tiedostot
  currentFile: string;           // Aktiivinen tiedosto
  cursorPosition: Position;      // Kursori
  imports: string[];             // Importit lähellä kursoria
  similarCode: CodeSnippet[];    // Samankaltainen koodi muualta reposta
  gitBlame: GitBlame[];          // Kuka kirjoitti mitäkin
}

RAG kooditietokantaan

  • Koodi jaetaan funktio- ja luokkatasoisiin chunkkeihin
  • Jokainen chunk upotetaan kommentteineen
  • Haku: käyttäjän koodi → samankaltainen koodi → konteksti

”Kysy dokumentista” -tyylinen käyttöliittymä

async function askDocument(question: string, documentId: string) {
  // 1. Hae relevantit tekstiosuudet
  const textChunks = await vectorDB.search(question, { documentId });

  // 2. Hae relevantit visuaaliset elementit
  const visuals = await vectorDB.searchVisual(question, { documentId });

  // 3. Yhdistä konteksti
  const context = buildMultimodalContext(textChunks, visuals);

  // 4. Generoi vastaus
  const answer = await multimodalLLM(question, context);

  return { answer, sources: [...textChunks, ...visuals] };
}

🎯Miksi tällä on väliä?

Miksi tällä on väliä? Viitearkkitehtuurit eivät ole kopioitavia reseptejä vaan lähtökohtia. Jokainen projekti on erilainen, mutta nämä esimerkit antavat sinulle käsityksen siitä, mitä komponentteja tarvitset ja miten ne voivat liittyä toisiinsa. Kopioi ideat, mukauta tarpeisiisi.
