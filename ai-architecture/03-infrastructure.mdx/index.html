3.3 MLOps ja LLMOps

AI-järjestelmän ylläpito on erilaista kuin perinteisen softan.

Dataputket

Keräys → Puhdistus → Versiointi → Validointi → Käyttöön

Jokaisen vaiheen pitää olla:

  • Toistettava — sama data → sama tulos aina
  • Versioitu — tiedetään millä datalla malli on koulutettu
  • Monitoroitu — tiedetään jos datan laatu heikkenee

Mallirekisterit ja elinkaarihallinta

# Mallirekisterimerkintä (MLflow)
model:
  name: "asiakaspalvelu-agentti"
  version: 3
  stage: "production"  # staging, production, archived
  metrics:
    accuracy: 0.94
    latency_ms: 450
  data_hash: "abc123def456"

// Ajautumisen tunnistus
function detectDrift(baseline: number[], current: number[]) {
  const statistic = kolmogorovSmirnovTest(baseline, current);
  if (statistic.pValue < 0.05) {
    alertTeam("Data drift detected!");
  }
}

3.4 Kustannusten hallinta ja optimointi

AI-sovellusten suurin menoerä on usein ei itse kehitys vaan ajokustannukset.

Token-pohjainen hinnoittelu ja budjetointi

function estimateCost(tokens: number, model: string) {
  const rates = {
    "gpt-4o": { input: 0.01, output: 0.03 },  // $/1K tokens
    "claude-3.5": { input: 0.008, output: 0.024 },
  };
  const rate = rates[model];
  return (tokens * rate.output) / 1000;
}

Välimuististrategiat

StrategiaSäästöEsimerkki

Tarkka välimuisti10–20%Täsmälleen sama kysely
Vastausten välimuisti5–15%CDN, selaimen cache

Pienempien mallien ketjutus vs. yksi iso malli

// Kalliimpi: yksi iso malli kaikkeen
const answer = await gpt4o(prompt);

// Halvempi: pieni malli seuloo, iso malli vastaa tarpeen mukaan
const isComplex = await gpt4oMini(`Onko tämä monimutkainen? ${prompt}`);
if (isComplex) {
  return gpt4o(prompt);
}
return gpt4oMini(prompt);

🎯Miksi tällä on väliä?

Miksi tällä on väliä? Infrastruktuuri määrittää, pääseekö AI-sovellus koskaan tuotantoon. 90% projekteista kaatuu siihen, että malli toimii kehittäjän koneella mutta ei tuotannossa. MLOps, kustannusten hallinta ja oikea inferenssiarkkitehtuuri ovat yhtä tärkeitä kuin itse malli.
