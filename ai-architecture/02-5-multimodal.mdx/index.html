Luku 6 — Arkkitehtuurityypit

Multimodaaliset arkkitehtuurit

Kuva+teksti, video+ääni+teksti -putket ja yhteinen upotusavaruus.

Kuva+teksti, video+ääni+teksti -putket

Multimodaalinen AI tarkoittaa järjestelmiä, jotka käsittelevät ja yhdistävät useita eri datatyyppejä: tekstiä, kuvia, ääntä, videota ja taulukoita.

📐

Kaavio: multimodal-pipeline

Interaktiivinen diagrammi tulossa vaiheessa 3.

Erilliset enkooderit + yhteinen upotusavaruus

Multimodaalisen järjestelmän ydinidea on yhteinen upotusavaruus (joint embedding space). Jokainen datatyyppi projisoidaan samaan vektoriavaruuteen, jossa samankaltaiset asiat ovat lähellä toisiaan.

  • Teksti: “koira juoksee puistossa” → [0.2, 0.8, -0.3, …]
  • Kuva: valokuva koirasta → [0.25, 0.75, -0.28, …]
  • Ääni: koiran haukunta → [0.22, 0.78, -0.31, …]
ℹ️

Avaintekniikka: CLIP (Contrastive Language-Image Pre-training) on tunnetuin menetelmä. Se kouluttaa tekstienkooderin ja kuvienkooderin rinnakkain siten, että oikea kuva-teksti -pari on lähellä toisiaan ja väärät parit kaukana toisistaan.

💡 Reaalimaailman analogia

Analogia: Multimodaalinen järjestelmä on kuin tulkkikeskus, jossa on eri kielten tulkkeja. Kun joku puhuu suomea (teksti), suomentulkki kääntää sen yhteiskielelle (upotusavaruus). Kun toinen näyttää kuvan (kuva), kuvatulkki kääntää myös sen yhteiskielelle. Nyt molemmat voivat “keskustella” samassa kielessä, vaikka heidän alkuperäiset kielensä ovat eri.

Samalla tavalla teksti, kuvat ja ääni muutetaan yhteiseen “konekieleen”, jossa niitä voi verrata ja yhdistellä.

Käyttötapaukset

Dokumenttianalyysi

Skannattu PDF → OCR → taulukoiden irrotus → semanttinen indeksointi → “kysy dokumentilta”

async function analyzeDocument(imageBase64: string, question: string) {
  const response = await multimodalModel({
    images: [imageBase64],
    prompt: `Analysoi tämä dokumentti. ${question}`,
  });
  return response;
}

Videoanalyysi

Valitse avainruudut, analysoi ne multimodaalimallilla, yhdistä tulokset.

  1. Pura video ruuduiksi (1 frame/sekunti)
  2. Analysoi jokainen ruutu erikseen
  3. Yhdistä analyysit aikajanaksi
  4. Vastaa kysymyksiin videon sisällöstä

Reaaliaikainen tulkinta

Kamera + mikrofoni → reaaliaikainen analyysi → ääniohjaus:

Käyttäjä (näyttää kuvaa): "Mitä tässä on?"
AI: "Kuvassa on kolme henkilöä kokoushuoneessa, valkotaulu täynnä muistiinpanoja"
Käyttäjä: "Tee niistä tiivistelmä"
AI (analysoi taulun tekstit): "Kokouksessa käsiteltiin..."

Tekninen toteutus

# Yksinkertainen multimodaalinen luokittelija CLIPillä
import torch
from transformers import CLIPProcessor, CLIPModel

model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32")
processor = CLIPProcessor.from_pretrained("openai/clip-vit-base-patch32")

def classify_image(image, labels):
    inputs = processor(
        text=labels,
        images=image,
        return_tensors="pt",
        padding=True
    )
    outputs = model(**inputs)
    logits_per_image = outputs.logits_per_image
    probs = logits_per_image.softmax(dim=1)
    return labels[probs.argmax().item()]

🎯Miksi tällä on väliä?

Miksi tällä on väliä? Multimodaalisuus on seuraava iso askel AI-sovelluksissa. Yhä useampi käyttötapaus vaatii sekä tekstin, kuvien että äänen käsittelyä. Arkkitehtuurin on tuettava eri modaliteetteja joustavasti — ja ennen kaikkea oikealla hinnalla. Ei kannata lähettää kuvaa analysoitavaksi, jos teksti riittää.
