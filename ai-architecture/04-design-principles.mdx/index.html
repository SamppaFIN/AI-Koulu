Arkkitehtuuriset suunnitteluperiaatteet

Luotettavuus, skaalautuvuus, turvallisuus ja havainnoitavuus AI-järjestelmissä.

4.1 Luotettavuus ja virheensieto

AI-järjestelmät ovat perusluonteeltaan epädeterministisiä. Tämä tarkoittaa, että perinteiset virheenkäsittelymallit eivät toimi.

Epädeterminististen komponenttien käsittely

// Perinteinen: oleta että toimii
const result = api.call(data);
assert(result.isValid);

// AI-maailma: oleta että voi epäonnistua
const result = await ai.call(data);

// Tarkista ensin, luota sitten
if (!isValidResult(result)) {
  result = await fallbackStrategy(data);
}

Fallback-strategiat

Kun LLM epäonnistuu, sinulla pitää olla suunnitelma:

TasoStrategiaLatenssi
1Yritä uudelleen (retry)+1×
2Pienempi malli (gpt-4o → gpt-4o-mini)−50%
3Välimuistista (aiempi samankaltainen vastaus)−90%
4Sääntöpohjainen (template + parametrit)−99%
5Virheilmoitus käyttäjälle—
⚠️

Tärkeää: Fallback-strategia pitää testata ennen tuotantoonmenoa. Jos LLM epäonnistuu, tiedätkö mitä tapahtuu? Moni järjestelmä kaatuu hiljaa siksi, että fallbackia ei ole koskaan testattu.

Uudelleenyrityslogiikka ja piirikatkaisijat

// Piirikatkaisija (Circuit Breaker)
class CircuitBreaker {
  private failures = 0;
  private threshold = 5;
  private state: "closed" | "open" | "half-open" = "closed";

  async call(fn: () => Promise<any>) {
    if (this.state === "open") {
      throw new Error("Circuit breaker open");
    }
    try {
      const result = await fn();
      this.failures = 0;
      return result;
    } catch (e) {
      this.failures++;
      if (this.failures >= this.threshold) {
        this.state = "open";
        setTimeout(() => { this.state = "half-open"; }, 30000);
      }
      throw e;
    }
  }
}

”Defensive design” probabilistisessa maailmassa

  1. Älä luota mihinkään — tarkista jokainen vastaus
  2. Rajoita vahinkoja — minimoi mitä malli voi tehdä
  3. Oleta pahinta — malli hallusinoi, toimi sen mukaisesti

4.3 Turvallisuus ja vastuullinen AI

Arkkitehtuuritason suojamekanismit

// Syötevalidointi
function validateInput(input: string) {
  // 1. Estä jailbreak-yritykset
  if (containsJailbreak(input)) return reject();
  // 2. Poista PII
  input = removePII(input);
  // 3. Rajoita pituus

}

Kustannusperäinen havainnointi

-- Esimerkki: kalleimmat käyttäjät
SELECT
  user_id,
  SUM(tokens) as total_tokens,
  SUM(cost) as total_cost,
  COUNT(*) as request_count
FROM ai_logs
WHERE date = CURRENT_DATE
GROUP BY user_id
ORDER BY total_cost DESC
LIMIT 10;

🎯Miksi tällä on väliä?

Miksi tällä on väliä? Nämä neljä periaatetta — luotettavuus, skaalautuvuus, turvallisuus ja havainnoitavuus — ovat niitä, jotka erottavat leikkikalun tuotantojärjestelmästä. Yksikään näistä ei ole “kiva lisä” vaan välttämättömyys, jos järjestelmän on tarkoitus palvella oikeita käyttäjiä oikeassa maailmassa.
