## 4.1 Standardit ja vaatimukset

### WCAG 2.2 AA/AAA

WCAG (Web Content Accessibility Guidelines) on kansainvälinen standardi
verkkosisällön saavutettavuudelle. **AA-taso** on lakisääteinen minimi
useimmissa maissa (EU, USA, UK).

| Periaate | Kuvaus | Esimerkkejä |
|---|---|---|
| **Havaittava** | Sisältö on kaikkien aistittavissa | Tekstivastineet kuville, tekstitys videoille |
| **Toiminnallinen** | Kaikki toiminnot on käytettävissä | Näppäimistönavigaatio, riittävä aika |
| **Ymmärrettävä** | Sisältö ja käyttöliittymä on loogisia | Selkeä kieli, ennustettava toiminta |
| **Toimintavarma** | Yhteensopiva apuvälineiden kanssa | Semanttinen HTML, ARIA |

### EN 301 549 (EU)

EU:n saavutettavuusdirektiivi — **pakollinen** julkisille verkkosivuille
ja -palveluille EU-alueella.

### Ruudunlukijat

| Ruudunlukija | Alusta | Markkinaosuus |
|---|---|---|
| **NVDA** | Windows | ~40% (ilmainen, avoin) |
| **JAWS** | Windows | ~30% (maksullinen, vanhin) |
| **VoiceOver** | macOS, iOS | ~20% (sisäänrakennettu) |
| **TalkBack** | Android | ~10% (sisäänrakennettu) |

<Callout type="info">
  **Testausvinkki:** Laita ruudunlukija päälle ja sulje silmäsi.
  Pystytkö käyttämään sivustoa? Jos et, sinulla on saavutettavuusongelma.
</Callout>

<AnalogyBox>
  **Analogia:** Saavutettavuus on kuin rakennuksen esteettömyys.
  - **Luiskat** (tekstitys) auttavat pyörätuolikäyttäjiä — ja myös
    lastenvaunujen kanssa liikkuvia
  - **Hissti** (näppäimistönavigaatio) auttaa pyörätuolissa — ja myös
    kassien kanssa
  - **Selkeät opasteet** (semanttinen HTML) auttavat näkövammaisia — ja
    myös turisteja ja kiireisiä käyttäjiä

  Kukaan ei rakenna portaita ilman luiskaa — älä tee samaa verkkosivullasi.
</AnalogyBox>

## 4.2 Tekninen saavutettavuus

### Semanttinen HTML

```html
<!-- Huono: div-pohjainen -->
<div class="button" onclick="submit()">Lähetä</div>

<!-- Hyvä: natiivi -->
<button type="submit">Lähetä</button>
```

Natiivit elementit tulevat sisäänrakennetun saavutettavuuden kanssa —
älä keksi pyörää uudelleen.

### ARIA-attribuutit

ARIA (Accessible Rich Internet Applications) korjaa semantiikan, kun
natiivia elementtiä ei voi käyttää:

```html
<div
  role="button"
  tabindex="0"
  aria-label="Sulje ikkuna"
  onkeydown="if(event.key==='Enter') this.click()"
>
  ×
</div>
```

**ARIA-sääntö:** "No ARIA is better than bad ARIA." — Käytä vain kun
natiivi elementti ei riitä.

### Otsikkohierarkia ja maamerkit

```html
<header role="banner">
<nav role="navigation" aria-label="Päänavigaatio">
<main role="main">
<aside role="complementary">
<footer role="contentinfo">
```

### Väri ei ainoana informaationa

- Älä koskaan käytä *pelkkää* väriä kertomaan jotain
- Lisää aina teksti, ikoni tai kuvio värin rinnalle
- Esim: virheilmoitus = punainen reunus + "Virhe" teksti + ⚠️ ikoni

### ARIA-anti-patternit — älä tee näin

❌ **Div-painike ilman roolia**
```html
<!-- ÄLÄ: ruudunlukija ei tunnista tätä painikkeeksi -->
<div onclick="submit()">Lähetä</div>

<!-- TEE: natiivi painike -->
<button type="submit">Lähetä</button>
```

❌ **Näkymätön fokus**
```css
/* ÄLÄ: fokus katoaa, näppäimistökäyttäjä eksyy */
*:focus { outline: none; }

/* TEE: selkeä fokusindikaattori */
*:focus-visible { outline: 2px solid var(--color-brand); outline-offset: 2px; }
```

❌ **Ikoni ilman tekstivastinetta**
```html
<!-- ÄLÄ: ruudunlukija sanoo "some image" -->
<img src="check.svg" />

<!-- TEE: alt-teksti kertoo merkityksen -->
<img src="check.svg" alt="Vahvistettu" />
<!-- TAI: jos ikoni on koristeellinen -->
<img src="decoration.svg" alt="" role="presentation" />
```

### Näppäimistönavigaation kulku

Tyypillinen näppäimistönavigaatio modaalissa:

```
1. TAB → Avaa modal (fokus menee modalille)
2. TAB → Seuraava kenttä / painike
3. SHIFT+TAB → Edellinen kenttä / painike
4. ENTER → Valitse / lähetä
5. ESC → Sulje modal (palaa edelliseen fokukseen)
6. TAB → Kun modal on auki, fokus pysyy modalin sisällä (focus trap)
```

### Tekstin skaalautuvuus

Sivun pitää toimia 200% zoomilla — ilman vaakasuoraa skrollausta.
Testaa zoomaamalla selaimessa.

## 4.3 Kognitiivinen saavutettavuus

### Selkeä kieli

- Vältä ammattislangia, käytä arkikieltä
- Lyhyet virkkeet (max 20 sanaa)
- Yksi idea per kappale

### Johdonmukaisuus

Sama toiminto → sama ulkoasu. "Osta"-nappi on aina sininen, ei joskus vihreä.

### Virheiden ehkäisy

"Oletko varma, että haluat poistaa?" on hyvä. "Peruuta"-painike ennen
kuin on liian myöhäistä on vielä parempi.

### Aikarajojen joustavuus

Jos sivulla on aikaraja (istunto, tarjous), anna käyttäjälle:
1. mahdollisuus pidentää aikaa
2. varoitus ennen umpeutumista
3. tieto siitä, että aika on loppumassa

## 4.4 Saavutettavuuden testaus

### Automaattiset työkalut

| Työkalu | Käyttötarkoitus |
|---|---|
| **axe-core** | Parhaat säännöt, Playwright/CI-integraatio |
| **Lighthouse** | Nopea yleiskatsaus, osana Chrome DevToolsia |
| **WAVE** | Visuaalinen, näyttää ongelmat suoraan sivulla |

### Manuaalinen testaus

Automaattityökalut löytävät ~30% ongelmista. Loput vaativat ihmistä:
- Navigoi pelkällä näppäimistöllä (TAB, Enter, Escape, nuolet)
- Lue ruudunlukijalla (NVDA, VoiceOver)
- Testaa zoomaamalla 200%
- Testaa kontrastit fyysisellä etäisyydellä

#### Ruudunlukijatestaus NVDA:lla (Windows)

1. **Lataa NVDA:** https://www.nvaccess.org/ — ilmainen, avoin lähdekoodi
2. **Käynnistä:** Asenna ja käynnistä NVDA (oletuspikanäppäin: CTRL+ALT+N)
3. **Testaa:** Sulje silmäsi ja navigoi sivua TAB-näppäimellä
   - Kuuletko kaikki elementit? Tiedätkö missä olet?
   - Ymmärrätkö mitä kenttään pitää kirjoittaa?
   - Toimiiko kaikki näppäimistöllä?
4. **Lopeta:** NVDA + Q lopettaa

> **Tavoiteltu lopputulos:** Sivun pitäisi olla käytettävissä täysin ilman hiirtä
> ja ilman ruutua — pelkällä näppäimistöllä ja kuulokkeilla.

### Saavutettavuuden tarkistuslista

Ennen julkaisua, käy tämä läpi:

- [ ] **Näppäimistö:** TAB kiertää koko sivun loogisessa järjestyksessä
- [ ] **Fokus:** Jokaisella elementillä on näkyvä fokus-indikaattori
- [ ] **Ruudunlukija:** NVDA/VoiceOver lukee kaikki elementit oikein
- [ ] **Kontrasti:** WCAG AA 4.5:1 normaalille tekstille, 3:1 suurelle
- [ ] **Zoomaa 200%:** Sivu toimii ilman vaakasuoraa skrollausta
- [ ] **Väri:** Väriä ei käytetä ainoana informaation lähteenä
- [ ] **Alt-tekstit:** Kaikilla kuvilla on alt-teksti (tai role="presentation")
- [ ] **Otsikot:** Otsikkohierarkia on looginen (h1 → h2 → h3, ei hypyillä)
- [ ] **Lomakkeet:** Kaikilla kentillä on label
